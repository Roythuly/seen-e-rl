# Contributing

## 协作原则

- Docs-first：跨模块设计先写文档，再开始实现
- Contracts-first：公共数据结构与接口命名先写进 `contracts/`
- Evals-first：新增关键行为前先补评测策略与测试范围
- Rollback-aware：任何未来实现都需要明确回滚或降级路径

## 提交流程

1. 更新对应模块文档或 PRD/RFC/ADR
2. 如有公共接口变更，同步更新 `contracts/`
3. 更新 `tests/` 或 `evals/` 中的计划说明
4. 运行校验脚本
5. 再提交实现代码

## 首版仓库状态

本仓库当前仍处于 `v0.1` 文档阶段，因此：

- 允许提交模板、占位 schema、说明文档
- 不允许把占位文件伪装成可运行实现
- 任何实现开始前都应先冻结当前文档基线

## 评审重点

- 模块边界是否清晰
- 公共接口是否稳定
- 是否破坏 docs-first 结构
- 是否给出了测试和评测落点
- 是否说明风险、故障模式和扩展方向
